## 8. Project Roadmap & Milestones

This project is developed in 10 strictly sequential milestones.

No milestone may implement functionality assigned to a future milestone.

Each milestone must be completed, reviewed, and approved before the next begins.

Milestone 1 – Development Foundation
Milestone 2 – Authentication & Project Management
Milestone 3 – Repository Upload & Storage
Milestone 4 – Queue & Job Management
Milestone 5 – Repository Intelligence
Milestone 6 – AI Planning (LangGraph)
Milestone 7 – Secure Runtime Execution
Milestone 8 – Dynamic Security Testing
Milestone 9 – Reports & Real-Time Frontend
Milestone 10 – Production Hardening

## Forbidden Actions

The AI must never:

• implement future milestones
• change repository structure
• rename folders
• rename APIs
• change database ownership
• introduce new frameworks
• replace libraries
• create generic utility folders
• duplicate business logic
• bypass the queue
• expose MS2 directly to frontend

When uncertain:

STOP

Ask for clarification.
